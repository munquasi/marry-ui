Customer Sites

The customer sites tab helps the user to add or edit site details in a customer record. The customer site details helps the user to look for the details in the location corner.

 

Locations are also known as sites within ERP. The purpose of creating a Site/location is for linking it to a customer account as the bill to or ship to address. The linking of the site/location to the customer account is done separately, via the customer account screen.

Add Sites to Customer Record

A single customer can have multiple operational sites. These sites need to be added to the record for proper workflow.

 

To add a Site:

  1. Under the Sites tab click the Add Sites button to add the site details for the selected customer. The Create Site screen appears as shown.

 

 

  1. In the Location panel, search for the location on the map that need to be added as a site.


 

  1. The searched location will starts appearing on the map.

  2. Click on Copy Coordinates button to copy the coordinates from the map to the location section. The copied coordinates will be reflected in the Latitude and Longitude fields.

  3. Select the country from the Country drop down.

  4. Fill in the requisite and desired details as necessary. 

 

By Default the Address Effective Date field displays the current date, you can change the date from the calendar attached.

 

  1. Click Save to save the changes after filling in the details.

  2. A pop up will appear on the screen displaying the name of the site being marked as saved.

Edit Sites of Customer Record

Whenever any site information gets changed, then the site record of the customer may need to be updated for smooth and error free processing of contracts. The site record in edit mode has the following sections:

  • Location (location of the site on a map)

  • Coordinates of the site

  • AI's / Surveys: Survey record/details for the site

  • Barred Employees: List of employees barred from working for the site

  • Address Details of the site

 

To edit a Site:

  1. Under the Sites tab click on any of the sites listed that you wish to edit as highlighted below.

 

The details of the site record gets displayed.

 

  1. Click the Edit button to make changes to the details.

  1. Enter the AI’s / Survey details.  User can maintain Survey related documents and information under the AI’s Site Survey section. The descriptions of the fields in AI’s / Surveys Customer are given below:

 

Field

Description

Document Name

Accepts the Survey Document name

Type 

User can pick two Survey types from the drop down list

1.     Site Survey

2.     Assignments Instructions

Status

Shows the status of the Assignments Instructions.

 

A drop down to pick the values from Accepted or Imposed will appear as status when you select Assignments Instructions as document type.

 

No status will appear when you select Survey in the type field.

Effective Date

The date on which the survey was conducted.

You can also type the date in DD/MM/YYYY format.

Expiry Period

Date the survey will expire. Select any one from the dropdown Values:

1.     3 Months

2.     12 Months

By default the Expiry Period will be 3months.

Location

Text field to hold the path / URL to the survey document stored.

 Expiry Date

 Date till which the Survey is Valid.

The Expiry date is calculated from the Effective Date till the value selected at Expiry Date (3 months & 12 Months).

A user cannot override the expiry date.

  1. Click Save against the respective edited section to save the changes.

  1. Click on Update on the parent screen once the required changes have been satisfactorily made.
    The site record will get updated and will appear in the list under the Sites tab.
