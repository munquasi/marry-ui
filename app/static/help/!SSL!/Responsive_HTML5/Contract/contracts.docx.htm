Ops-2.5 Work Order

A work order is a purchaseorder document which details the specific service or product that a customer intends to purchase from G4S. It is prepared with all the requirements of the customer and consists of information on the billing pattern, billing address, revenue plan and so on.

 

A work order is a partof a contract (which can consist of multiple work orders).

 

Purchase Order Information is also included in the Billing Information Section of a work order.

 

The Work order tab consists of the following segmented information:

●      Billing Information

●      Skills

●      Licenses

●      Equipments

●      Associated Contact

●      Location

●      Audit Log

●      Public Holidays

●      Invoice/Credits

 

The above segment based information appearing under the work order are inherited from the main Contract details.

The descriptions of the fields on the Work Order screen are given below:

Field

Description

Work Order Name

Name by which  the work order is visually identified

Business Unit

The business unit for which the work order is applicable

Legal Entity

The legal entity associated with this work order.

Organization

The organization that will service the sales order

The mode by which the proof of duty by the employee will be captured by the system. Displays IVR by default.

Start Date

Displays date from which the work order will be applicable.

End Date

Displays the date on which the work order will end.

Open Ended

Enabling this option means that there is not definite decided end date for this work order .

Ops-2.5.3.8.4 View/Manage Barred Employees for a Location

 

When a user has barred an employee from a site through a customer location record or from Site Record, then the employee is also barred from any new customer location record created for the same customer for the same site.

 

 

 

Therefore if new work order is raised to deliver services to the same site, these would automatically be reflected in any Customer location record assigned to that site and to the Site barred employee also.

 

So Barred Employees are visible and manageable at both the site record as well as the Customer location.
