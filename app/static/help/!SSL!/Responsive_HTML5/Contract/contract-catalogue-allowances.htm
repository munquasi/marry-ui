Allowances

To pay the G4S employees for the allowances it is very important to have the required allowance details in system and edit it whenever required. Based on these details, the employee will be paid and the client will be charged. Following are the steps to add and edit the Allowance details in the system.

 

You can perform the following tasks from this screen:

  1. Add Allowance

  2. Edit allowance

    Edit Allowances

    Any change done in the allowance tab will affect the billing for the employee according to the already set policies. The minimum/ maximum qualification hours can be changed whenever required. The Per Day allowance can be changed to Monthly/Weekly/per day according to the requirement.

     

    To edit an Allowance:

    1. On the list of customers displayed, click the desired customer record (highlighted in red). The existing account(s) related to the customer is displayed on the right side of the screen as (highlighted in blue) below:

     

     

    1. Click on the account link highlighted in red (shown in the preceding figure). The details existing under the customer account just clicked is displayed in tabbed view.

     

    1. Click on Contract tab to display the contract screen with the list of existing contracts (if any) as shown.

     

     

    1. Click on any of the existing contracts to view the details of the contract.

     

     

    1. Click the Catalogue tab. The Catalogue screen will appear with the following tabs:

    ●      Rate Card

    ●      Allowance

    By default the system will show Rate Card tab.

    The descriptions of the fields in Add New  Allowance Screen are given below:

     

    Field

    Description

    Evaluation Period

    The way in which the Employee is given allowance.

     

    The Allowance Amount is applicable for this Evaluation Period.

    Allowance Name

    The name or description of the allowance.

     

    The list should be fetched from the country module and displays only those allowances which are saved in active state in country module.

    Amount

    The amount for the given allowance.

    Allowance Type

    The type of allowance being defined.

     

    It is fetched from the country module in which values will be:

    ●      HCM Skill Based

    ●      Generic

    ●      Time Based.

    Billed

    This value should indicate whether the given allowance should be billed or not billed to the customer.

    Paid

    This value should indicate whether the given allowance should be payed or not payed to the employee.

    Start Date

    Captures the start date of the allowance. It displays the current date by default on the Start date field.

     

    To change/select another start date for the Allowance, click on the icon, the calendar to select date will get displayed as shown below

     

    Once the user select the start date, it will get displayed on the Start Date field.

    End Date

    Captures the end date for the allowance.

     

    To change/select another end date for the Allowance, click on the icon, the calendar to select date will get displayed as shown below:

     

    Once the user select the end date, it will get displayed on the End Date field.

    Pay Type

    Explains how the payment is done.

     

    To change the Pay Type, select the appropriate option from the drop down list:

    ●      Lump sum

    ●      Hourly

    Days

    Put a checkmark next to the days of the week for which the allowance need to be given.

    Start Time

    The start time of the particular day at which the allowance will be given.

    End Time

    The end time of the particular day till which the allowance will be given.

    Minimum Qualification Hours

     This field indicates the minimum time for which the employee has to work to become eligible to get an allowance.

    Maximum Qualification Hours

    The maximum hours count for which allowance need to be payed and/or billed.

    Pay Qualification Time

    This field should capture time to pay for allowance. If this indicator is false, then employee would be payed for allowance (if it fulfilled the minimum qualification time and exceeded the duty.)

     

    (for example.

    If Total hours worked by employee = 22 hours ,

    Minimum Qualification Time = 20 hours

    Then employee would be payed for 2 hours).

     

    If the indicator is true, then employee will be payed completely. As per above example allowance will be paid for 22 hours accordingly.

     

    (the user need to be very careful for this field as this field will reflect in the payment of the employee and the billing for the client)

    Skills

    This data comes from Reference data from HCM which will be maintained in country module for the allowance name selected.

    Expenditure Code

    This is the alphanumeric code which is fetched from finance department during costing procedure.

     

    Allowance Type, Expenditure Code and Skill gets automatically selected according to the selected Allowance Name.

     

    1. Click on Save once all the required details are entered.

    The following pop-up will appear.

Billing for Allowances

It is very important to pay the employees for the hard work and extra hours put in their respective jobs. The Allowance Tab (under Catalogue tab of Contract level) is used to store the information required to pay the employee and charge the client for specific allowances associated within the contract. These Allowances are first agreed with the client and then are able to be charged back to the client.

 

User can add a new allowance for the contract. User can also edit or delete the existing allowance with this tab.

 

Please refer section Add new allowance to know the steps to add an allowance and Edit allowance to know the steps to edit an allowance.

As soon as billing for a selected allowance is processed in the OPS system, the events are sent into PPM (Project Portfolio Management). Then PPM schedules an automatic process to transfer the PPM invoice to Accounts Receivables. Then in Accounts Receivables there is a process scheduled to automatically import the invoice from PPM, then invoice is sent to the customer.

Assigning an Allowance:

Following are the steps required to assign allowance under the Post tab of Work Order Location.

 

  1. Select the Work Order under the Contract for which the billing is required

 

     

 

  1. Click on the Location tab under the Work Order summary.

The following screen will appear on the screen.

 

    

 

  1. Click on Location and select any location.

 

    

 

    The following screen will appear.

 

         

 

Location needs to be Active to initiate the billing process.

 

  1. Click on the Post Tab to get the following screen.

 

 

  1. Select the post for which you want to add allowance. The following screen will appear.

 

    

 

  1. Click on the Allowance Tab to get the following screen.

 

   

 

This list of allowance is fetched from the allowances added in the catalogue tab at contract level.

 

  1. Put a checkmark under the Action column of the required allowance that need to added to initiate its billing process.

    The following pop up screen will appear.

 

    

 

Allowance can be added on for a draft post.

 

  1. Click OK to add the selected allowance.

    The selected allowance will be displayed on the screen and the process for billing will start automatically.
