Work Orders

Work order are orders from customers which details the specific service or product that the customer intends to purchase from G4S. It is prepared with all the requirements of the customer and consists of information on the billing pattern, billing address, revenue plan and so on.

 

A work order is a part of a contract (which can consist of multiple work orders).

 

Purchase Order Information is also included in the Billing Information Section of a work order.

 

The Work order tab consists of the following segmented information:

●      Billing Information

●      Skills

●      Licenses

●      Equipments

●      Associated Contact

●      Location

●      Audit Log

●      Public Holidays

●      Invoice/Credits

 

The above segment based information appearing under the work order are inherited from the main Contract details.

Ops-2.5.1 Add Work Order

A contract Manager can create a work order here. Following are the steps to add work order in the system:

1.     Search for and open an existing contract (refer previous section- “Searching for a contract”), OR successfully create a new contract under a customer account (refer previous section- “Creating a contract”). The following screen displays:

2.     Scroll down the screen and click the Work Order tab. The following screen displays:

3.     Click the  button. The following screen displays.

4.     Select or enter the required fields in the screen.

5.     Click the Save button to save the new Work Order record.

 

The descriptions of the fields on the Work Order screen are given below:

 

Field

Description

Work Order Name

Name by which  the work order is visually identified

Business Unit

The business unit for which the work order is applicable

Legal Entity

The legal entity associated with this work order.

Organization

The organization that will service the sales order

Proof Of Duty

The mode by which the proof of duty by the employee will be captured by the system. Displays IVR by default.

 

Once the user select any of the option from the drop down, the information will get displayed on the Proof Of Duty field.

Bill Plan

This is the Bill Plan  applicable for this work order.

Once the user is able to search the required Bill Plan, it will get displayed on the Bill Plan field.

Revenue Plan

It is the revenue plan that will be considered for this work order.

 

Displays the last clicked Revenue Plan by default.

 

Once the user select any of the option from the drop down, the information will get displayed on the Revenue Plan field.

Start Date

Displays date from which the work order will be applicable.

End Date

Displays the date on which the work order will end.

Open Ended

Enabling this option means that there is not definite decided end date for this work order .
