Ops-6-6 System Administration

OPS- 6-6.1 Create a Role

In the Create Roles screen in System Administration tab of Javelin OPS interface, the System Administrator can create and edit roles and manage the following:

 

●    Role Name (e.g. Contract Manager)

●    Role Description (this should describe the permissions which this role has)

●    Role Comments (the comments can be used for any important information, including for example a change request number requesting the role to be created, updated, suspended)

 

To create a new Role:

 

1.     Click icon and then select the Create Roles option.

 

2.     Next, click New on the resulting screen.

3.     On the resulting screen, click New. The Create a new role screen gets displayed.

4.     Enter the role name, description, and comments (if any), and click Save to save the new role.

 

In case the User is not mapped to an Employee or if for any reason the application is unable to retrieve the Employee’s Job, the user will automatically be granted the Least Privileged user’s role. 

OPS- 6-6.2 Update a Role

The user can update an existing role if required. The right to edit the fields depend on the privilege given to the administrative user.

 

For the System Administrator and Least Privilege User roles, only the Comments field can be updated.

1.     Click System Administration and then select the Create Roles option.

2.     Next, click on any of the existing roles on the resulting screen.

5.     Click Confirm after making the required changes to save the changes.

OPS- 6-6.5 Mapping Jobs to Role

Jobs are mastered in Oracle HCM and stored in the Employee module in OPS. In the Roles module, we utilise this reference data and display the list of Jobs in the Job Title to Role Mapping screen.

 

Here, the System Administrator can map a Job Title to a Role. This allows a User to inherit a Role automatically based on their Primary Job Title in HCM.

 

To map a job title to a Role:

 

1.     Click icon and then select the Job Title to Role Mapping menu.

2.     The following screen is displayed on clicking the Job Title to Role Mapping menu.

3.     Click the link against the job title that you want to assign a role. The record expands with the various fields that allows the logged in user to update the required information:

 

4.     From the Role drop down list, select the appropriate role you want to assign to the selected job profile.

5.     After selecting the role to be mapped to the job profile, click Update to save the mapping. The following confirmation dialog displays.

6.     Click Confirm.The role selected will be assigned to the job profile.

 

OPS- 6-6.6 Update a Job to Role Mapping

The System Administrator can update the mapping of a new role from the Least Privileged User to another role.

 

If a Job Title is updated in HCM in the reference list of Job Titles, then that Job ID will be remapped to the Least Privileged User role automatically. The System Administrator would need to then log into the system and update the mapping to the relevant role.

 

To update job title to role mapping:

1.     Click icon and then select the Job Title to Role Mapping menu.

 

2.     The following screen is displayed on clicking the Job Title to Role Mapping option.

3.     Click the  icon against a record that belongs to the least privileged user. A screen displays with various fields that allows the logged in user to update the required role.

4.     Select the required role to be mapped. The user can also edit the comments (if required). Click Update. The following confirmation message displays.

 

The System Administrator can update the mapping from Least Privileged User to another role, (which was created on the Create Roles page).

 

5.     Click Confirm. The role selected will be updated for the job profile.

OPS- 6-6.7 Suspending a Job Title

The System Administrator cannot suspend a job title in OPS as Job Titles are mastered in HCM. If a Job Title is suspended in HCM, then it will also be suspended in OPS.

 

OPS- 6-6.8 Configuring/Updating Menus For Role

The System Administrator can view the list of available menus in the user interface and configure for each role, the menu items to be shown in the left hand bar in the Javelin OPS application.

 

Then by selecting the checkboxes next to each of the menus you will grant or remove access to these menus for this role.

 

To Configure/update Menus for a Role:

1.     Click icon and then select the Roles Administration:Menus menu.

2.     The following screen is displayed on clicking the Roles Administration: Menus option.

 

3.    From the Roles drop down list: select the role whose menu you would like to configure.

 

4.     For the selected Role, the list of menu items are displayed against which the logged in user can enable or disable the access.

 

Menus and menu items are configured in a parent / child relationship. When the parent is checked, the child items will automatically be checked, unless otherwise overridden.

 

 

5.     Then by selecting the checkboxes next to each of the menu items listed, you can grant or remove access to these menus for this role.

6.     Click the Save button to confirm the changes.

 

Once a change has been made you must click Save in the top left hand side of the screen to save the changes.

 

When you select a Role and then begin to make changes to the menu permissions, an Undo Changes button will appear. This is so that BEFORE you press save, it is possible to restore the permissions to the previously saved version.

Note: once you press save, the Undo Changes is no longer possible.

OPS- 6-6.9 Role Administration in CRUD

The System Administrator role can access the role administration page and manage the CRUD (Create, Read, Update and Delete) operations of all the users coming under the particular user role selected by the admin and change access to operations under the country.

 

The CRUD functions are the user interfaces to databases permitting users to create, view, and alter data. CRUD works on and manipulates entities in databases.

 

 

1.    Click  icon and then select Roles Administration: Menus (CRUD) menu.

            

2.     The following screen gets displayed on clicking Roles Administration: Menus (CRUD) menu.

3.     Select the required role from the Role drop down list.

4.     For the selected Role, the list of menu items are displayed against which the logged in user can enable or disable the access.

 

You can click Expand All option to expand the root modules to their submenus. Collapse all does the reverse.

 

5.     Select the checkboxes next to each of the menus, which you want to can grant or remove access for this role.

 

Menus and menu items are configured in a parent / child relationship. When the parent is checked, the child items will automatically be checked, unless otherwise overridden.

 

 

Once a change has been made you must click Save in the top left hand side of the screen to save the changes.

 

When you select a Role and then begin to make changes to the menu permissions, an Undo Changes button will appear. This is so that BEFORE you press save, it is possible to restore the permissions to the previously saved version.

Note: once you press save, the Undo Changes is no longer possible.

 

6.     Click Save to proceed with the changes, or Undo Changes to cancel the changes.
