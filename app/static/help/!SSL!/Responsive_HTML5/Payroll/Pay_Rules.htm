Pay Rules

The Pay Rules module allows you to manage the rules of salary generation of the various groups of employees. These rules will be applied while generating payments for the employees. You can attach pay rules defined in the system with a pay profile for processing the pay according to that attached pay rule (s).

 

Create Pay Rule

User can create a pay rule with multiple conditions so that the system can generate the correct pay for employees working in a specific shift.

 

To create a pay rule:

 

  1. Click on the Payroll module and then select Pay Rules option

A list of existing pay rules are displayed. 

 

    

 

  1. Click the Create New Rule button to create a new pay rule.

 

    

 

  1. Fill the required and relevant details, and click Save.

    The description of the fields are as follows.

 

Field/Option

Description

Name

The name by which the Pay Rule is visually identified.

Expenditure Type

Used to map pay elements with expenditure types for Finance/ERP cost allocations (Basic Pay, Holiday pay, Overtime pay, Sick Pay). This information is populated by what is set in the country module the same manner as Site allowances.

Rate Type

Type of rate applicable for this pay rule (Hourly rate, Multiplier, Lump sum).

Rate Amount

Value of rate

Pay Element ID

This field helps to identify the type of payment incurred (whether it is for holiday, overtime, etc). This list is created in the country module.

Core Basic Pay

Indicates this is a core basic and will cause pay conditions to be predefined in order to ensure all the basic hours an employee works are accounted for.

Include in Average Pay calculation for absences

If check-marked, this payrule will be included in average pay calculation for absences.

Effective From Date

Date the pay rule will be effective from.

Pay Rate

Rate of pay

Do not Pay if in Probation Period

The Employee will not be paid if he/she is on probation.

Rate % Modifier

This option allows you to apply a probation rate to the basic employee rate while they are in their probation period.

Hours Paid

(Contracted hours, Scheduled Hours, Average Hours).

 

 

Field/Option

Description

Time of Day

The time of days that some of the shift must fall within in order for this rule to be applied. Only the portion of the shift that falls within these times will contribute toward gross pay entry quantity.

Normal Days of Week

The days of the week that some of the shift must fall within in order for this rule to be applied. Only the portion of the shift that falls on these days will contribute toward gross pay entry quantity.

Bank Holiday Days

The days of the week that some of the shift must fall within that are also bank holidays in order for this rule to be applied. Only the portion of the shift that falls on these days will contribute toward gross pay entry quantity.

Recurring

The pay run will recur regularly to pay the group of employees

Set Specific Dates

This field will be used to create rule such as active shift on a  specific date (such as a special holiday). This rule gets triggered only if the shift is done on the date mentioned in this field.

Some allowances only get paid on a holiday (as specified by that country or contract) that are over and above the holidays already defined in the standard calendar. These are known as 'special holidays'. The pay rule will look to the special holiday set in the calendar to determine if the allowance should be paid. In Pay profile, there is a field 'Bank Holiday' which is used to select calendar of a particular country.

Period

It is the pay period being considered - Hourly/Quarterly/Annually/Pay Period.

Number of consecutive days required

Number  of Consecutive Days required before this rule is applied. This is the minimum number of working days that has to be recorded consecutively in order for the rule to trigger.

Range of Days

It is the range of days that will be considered as belonging to this shift.

Minimum Qualifying Hours

If the user has entered any value in this field, then this rule will be only triggered for shifts meeting the minimum hours required.

Maximum Qualifying Hours

If the user has entered any value in this field, then this rule will be only triggered for shifts meeting the maximum hours required.

 

Field/Option

Description

Assignment Type

The type of assignment (Regular, holiday, sickness, absence)

Time Type

The time being given by the Employee (Overtime or basic)

Min Hours Paid

If this rule is applicable to a shift, the employee will get paid at least for hours defined here even if he has worked less than the hours defined here.

Applicable Hours

In this case, this particular shift is applicable to the length of shift defined in the rule.

Employee

This field allows you to associate multiple employees with the Pay rule.

 

 

If the user has set the period to weekly, then on a weekly basis, the system will check all the shifts to determine if the minimum qualifying time is exceeded- if yes, it will apply the rule.

A) If the Rate Type in the rule is Lump sum, it will be applied once.

B) If the Rate Type in the rule is  hourly rate or multiplier, it will be applied from minimum hours up to maximum qualifying hours specified.

 

 

  1. After you have filled in all the relevant fields according to your requirement, click Submit on the upper right corner of the screen.

    After the user has created the pay rules, these pay rules can be used in the pay profile while creating or updating a pay profile.

 

Edit Existing Pay Rule

From this screen, you can make certain changes to the Pay rule details as required.

 

To edit an existing pay rule:

 

  1. Click on the Payroll Administration module and then select Pay Rules option.

          A list of existing Pay rules gets displayed.

 

    

 

  1. Click on any of the desired pay rule to view details of that pay rule.

 

    

 

 

  1. Make the required changes to the pay rule.

  1. Click Save at the top right corner of the screen to store the changes to the pay rule.

 

Whenever any changes are made to payroll, pay profile or pay rule, the amendments will get reflected in the payroll file and it will be part of the pay calculations.

 

 

When a payroll run currently being calculated and the user attempts to update a rule that is included in the gross pay calculation, then the rule gets locked during the calculation. In this case the user gets a pop-up message stating that the rule cannot be edited.
