Payroll

The payroll module include payroll administration function that allows you to create and configure payrolls, configure pay rules, facilitate processing of payrolls, running of payrolls, set payroll frequency, configure pay profiles, view a dashboard of payroll runs etc.

 

This module helps manage and organize the compensation of employees by tracking totals hours worked, rates of pay, allowances, conditions for working hours and so on. It archives employee payment records giving you a concrete payment history. The payroll module facilitates payment of your employees on time according to the parameters that you set, leaving little margin for human error.

 

Payroll administration also makes sure that company payroll policies adhere to employment laws.

 

  • The Payroll Module provides the following features among others:

  • Payroll Administration

  • Manage Pay Profiles

  • Set Pay Rules

  • Manage One off Payroll

  • View and Manage Payroll run Dashboard

  • Manage Minimum wages

 

 

Create New Payroll

You can create a payroll which can be configured to run on a periodic basis (Monthly, fortnightly, weekly). When it runs on the scheduled time, it will calculate the salaries, allowances, bonuses, overtime automatically based on the pay rules linked to the pay profiles attached to that pay roll.

 

To Create a Payroll:

 

  1. Click on the Payroll Administration module and then select Pay Roll option.

 

  1. Click the Create New Payroll button to create a new payroll.

 

    

 

  1. Fill in the desired payroll details as required.

 

    

 

Following are the description of the fields present in the Add Payroll screen:

 

Field/Option

Description

Days after end day

Number of days after the pay period end when the Payroll XML file will be generated.

Time of Day

Time of the day when the Payroll XML file will be generated.

First run date

Date after which the payroll schedule will begin.

Last run date

Date when the last payroll XML file generation will occur.

Effective from Date

Date from when the payroll will be effective.

Scheduled

Whether the Payroll is scheduled or not.

Ad-hoc Payroll Run Date

It is the payroll run date as decided at the time of creation, which is subject to revision as per requirement.

 

 

Field/Option

Description

Name

Name by which the period will be visually identified

Pay Period

The duration of the period each payroll run will cover

Shift Straddle Rule

Determines how to handle shifts that straddle the start/end dates of the pay period.

●      Shift Straddle: For example, on the last day of the payroll period end date, a shift might straddle over midnight into the next pay period. The system apportions the hours into the correct pay period.

○      e.g shift = 19:00-07:00 5th Oct -6th Oct

○      Pay period = monthly, paid on 5th of month

○      19:00-23:59 = October Pay Period

○      00:00-07:00 = November Pay Period

●      Shifts that ends in the period: For example, if period end date is 31st August and there is a shift that has a start time on 31st July but ends on the 1st August, then that whole shift will be moved to the August pay period

or

●      If period end date is 31st August and there is a shift that has a start time on 31st August, but ends on the 1st September, then that whole shift will be shifted the Sept pay period.

Pay Profiles

Indicates which pay profiles are associated to this payroll. Only pay profile with matching pay period frequency/duration can be linked. A pay profile can only be linked to one payroll and a payroll may have 1 or more pay profiles linked.

 

To set the Pay period for a Payroll:

 

  1. Provide the name of the payroll being created and set the Pay Period as monthly, fortnightly or weekly.

 

  1. Select the Shift Straddle rule which can be:

  • Split shift

  • Shifts that start in the period

  • Shifts that end in the period

 

  1. In case you select weekly or fortnightly, the Weekly Start Day and Weekly End Day will be displayed which you need to select.

 

To configure Payroll run Rule:

 

  1. In the Payroll Run section, enter the Days after End day information.

For example if you enter 2 here, the payroll will be run 2 days after the end of the pay period.

 

    

 

  1. Enter the Time of day when the Payroll will be run.

 

  1. You have up to two run dates, a first run date and a second run date for the payroll.

 

  1. The Effective from Date allows you to set the date from which the payroll run is being considered.

 

  1. You can enable the Scheduled  check box to enable the schedules you have filled in for this payroll.

 

Payroll once run will come under the past payroll in the payroll run dashboard.

 

Attach Pay Profile to a Payroll

The user needs to map or attach a pay profile to a payroll so that when that payroll is run or processed, the pay profile (and the pay rule associated with it) will be used in generating the pay file. A pay profile can only be linked to one payroll and a payroll may have one or more pay profiles linked to it.

 

This explanation assumes that you have already created pay profiles. For steps on how to create or view existing Pay profiles, refer to OPS-6.2 Pay Profiles.

 

To attach a pay profile to a payroll:

 

  1. Click on the Payroll Administration module and then select PayRoll option.

 

  1. Click the Create New Payroll button to create a new payroll.

 

    

 

  1. Fill in the desired payroll details as required.

 

  1. In the Pay Profile section, select a pay profile from the list that will be used while running this payroll.

 

    

 

  1. Click Save after selecting the desired Pay profile for the payroll.

    The selected Pay profile will be included in the newly created Payroll .

    The Pay Profile when included in a payroll will use the pay rules included in that pay  profile while calculating the pay.

To view details of an existing Payroll:

  1. You can filter your view of the records by typing in the keywords of your search criteria.

 

    

 

  1. Click on any of the payroll record (as shown below) to view the details of that payroll.

 

    

 

  1. The details of that payroll will be displayed as shown below.

 

    

 

Run a Payroll

When you run a Payroll, the payroll system will calculate the pay for the employees for the current cycle, based on the period selected and produce a gross pay file which gets submitted to the finance department (payroll section) for processing. The history of all these payroll runs are maintained in the Payroll Run Dashboard under past payroll section.

 

To run or process a Payroll:

 

  1. Click on the Payroll Administration module and then select Pay Roll option.

          The list of existing payrolls are displayed as a list.

 

    

 

  1. Click on any of the payroll record (as shown above) to view the details of that payroll.

 

    

 

  1. Click the Run Payroll button to run the payroll.

 

After running a payroll, if the user still wants to make changes to the payroll data and re-process the payroll, the user can click the Payroll Process button to re-create the xml output file with corrected result.

 

Once a payroll has been run, the user can edit only a few fields such as Payroll Description, Associated pay profiles, Payroll run days (after period end), Payroll run time of day, Payroll run period start and End date.

 

Once the payroll run is completed and submitted to the Payroll/Finance, it shows in the Payroll dashboard in the Previous Payroll Runs. User can only view the previous payroll runs.

 

  1. Click the Save button to save the Payroll that has been generated in the latest payroll run.

 

  1. Click the Submit button to lock the Payroll that has been run so that changes to the pay file are no longer allowed.

 

Process a Payroll

When you want to re-process a payroll for a pay cycle and update any data if there has been any change, then you can process a payroll. The history of all these payroll runs are maintained in the Payroll Run Dashboard under past payroll section.

 

To process a Payroll:

 

  1. Click on the Payroll Administration module and then select Pay Roll option.

          The list of existing payrolls are displayed as a list.

 

    

 

  1. Click on any of the payroll record (as shown above) to view the details of that payroll.

  

    

 

  1. Click the Payroll Process button to re-process the payroll.

 

Once a payroll has been processed, changes and adjustments to the payroll will be re-processed and reflected in the xml file generated.

Once the payroll run is completed and submitted to the Payroll/Finance, it shows in the Payroll dashboard in the Previous Payroll Runs.

User can only view the previous payroll runs.

 

  1. Click the Save button to save the Payroll that has been generated in the latest payroll run.

 

  1. Click the Submit button to lock the Payroll that has been run so that changes to the pay file are no longer allowed.

 

Once you click the Submit button, the generated payroll will be locked.

 

Edit Payroll Details

For a payroll which has not been run till date, all the fields can be amended. However, for a payroll where payroll run has occurred even once, the user can amend only a few fields about that Payroll.

 

To edit the details of a payroll:

 

  1. Click on the Payroll Administration module and then select PayRoll option.

              The list of existing payrolls are displayed as a list.

 

    

 

  1. Click on any of the payroll record (as shown above) to view the details of that payroll.

 

    

 

Field/Option

Description

Days after end day

Number of days after the pay period end when the Payroll XML file will be generated.

 

  1. You can only alter the Days After End Day value while editing an existing Payroll.

     

  1. You can click the Save button to save the changes.

     

  1. You can click the Submit button after running the payroll and clicking Save. Submit action will lock the payroll from further changes.

 

Whenever any changes are made to payroll, pay profile or pay rule, the amendments will get reflected in the payroll file and it will be part of the pay calculations.

To set the payroll frequency:
